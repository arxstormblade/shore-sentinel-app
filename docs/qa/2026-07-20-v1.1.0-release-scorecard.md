## Release promotion rule

The release tag is created only at promotion: after the reviewed candidate passes CI, required QA/security gates, and staged deployment/rollback validation. Do not instruct operators to clone `v1.1.0` before that promotion has occurred; use the reviewed branch or immutable commit SHA instead.

## Temporary v1.1.0 risk acceptance

On 2026-07-20, the release owner explicitly waived the non-local SSH/egress validation matrix for v1.1.0 only: disposable-fixture SSH/ForceCommand/native-sudo/cancellation proof and deployed worker-egress firewall proof. This waiver does not mark those controls verified and does not waive staged Docker Compose health, browser/API critical-flow, log-review, or rollback evidence. The matrix must be completed before enabling or relying on managed remote scanning in a later release.
